The scope of var, let, and const

In JavaScript, 'var', 'let', and 'const' are used to declare variables. The scope of a variable refers to the part of the program where the variable is accessible.
'var' declarations are globally scoped or function/locally scoped. A variable declared with 'var' outside a function block is available for use in the whole window, while it is available and can be accessed only within that function when declared within a function.
'let' and 'const' are block-scoped, meaning they are only accessible within the block they are defined in. A variable declared with 'let' can be updated but not re-declared, while a variable declared with 'const' cannot be updated or re-declared.

The use cases of null and undefined

In JavaScript, 'null' and 'undefined' are two distinct types with different use cases.
undefined is a type that represents a variable that has been declared but has not been assigned a value. It can also be used to indicate that a variable does not exist.
On the other hand, null is an assignment value that can be assigned to a variable as a representation of no value. It is often used to indicate that an object does not exist or that a variable has no value.

What do you mean by REST API?

A REST API is an API that conforms to the design principles of the REST, or representational state transfer architectural style. It is a set of rules that define how applications or devices can connect to and communicate with each other. REST APIs are sometimes referred to as RESTful APIs
